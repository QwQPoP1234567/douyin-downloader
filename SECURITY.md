# 安全政策

## 支持范围

安全修复以最新的 `main` 分支为准。旧版本可能不会单独回溯维护。

## 报告漏洞

请不要在公开 Issue 中披露可直接利用的漏洞、Cookie、访问令牌、Webhook、数据库密码或个人账号信息。

优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告入口。报告请包含：

- 受影响版本或提交；
- 复现步骤；
- 实际影响；
- 建议修复方式；
- 已确认不包含真实用户 Cookie、密码和下载内容的最小复现材料。

## 敏感文件

以下内容永远不应提交到 Git：

- `.env`；
- `data/` 数据库；
- `browser_data/` 浏览器登录会话；
- `downloads/` 下载内容和元数据；
- 日志、崩溃转储和 Playwright 调试快照；
- 钉钉 Webhook、加签密钥和 GitHub Token。

如果敏感信息曾经进入 Git 历史，仅删除当前文件是不够的：应立即吊销密钥，并重写或放弃受污染的公开历史。

## 部署建议

- 管理页面默认仅监听本机；暴露到局域网前请配置防火墙。
- 不要把 Chrome CDP `9222`、VNC `5900` 或 noVNC `6080` 直接暴露到公网。
- Docker 初始化完成后关闭 `douyin-setup` 容器和 `8780` 端口。
- 为 `.env`、浏览器会话、MySQL 数据卷和下载目录设置最小必要权限。
